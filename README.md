# Water Level Dashboard

Water Level Dashboard is a modern, responsive web application for monitoring reservoir and river water levels across Indonesia in real-time. Built on top of Next.js 14, Tailwind CSS, PostgreSQL, and Prisma ORM.

## Tech Stack

- **Framework**: Next.js 14 (App Router)
- **Styling**: Tailwind CSS
- **Authentication**: NextAuth.js (Credentials Provider)
- **Database**: PostgreSQL
- **ORM**: Prisma (v5)
- **Maps**: OpenStreetMap integration via embedded iframes

## Features

- **Secure Authentication**: Robust login system with NextAuth connected to PostgreSQL.
- **Dynamic Dashboard**: Overview of system statistics including active sensors and critical alerts.
- **Waterlevel Monitoring**: 
  - Toggle between intuitive Card View and detailed Table View.
  - Interactive search and filtering by location, ID, or status.
  - Interactive maps pointing to exact real-world locations (Katulampa, Jatiluhur, Ciliwung, dll.).
  - **Database Integration**: Data is actively fetched server-side from PostgreSQL.
- **User Management**: Full CRUD operations for managing Administrators and standard Users.
- **Profile & App Settings**: 
  - Customize Application Name and Logo (persists across Sidebar, Header, and Login Page).
  - Update user profile information and password.
- **Mobile Responsive Layout**: Features robust grid constraints and horizontal scrolling tables to ensure zero text-overlap on mobile screens.

## Getting Started

### Prerequisites

- Node.js 18+
- PostgreSQL database running locally or remotely

### Installation

1. Clone the repository and install dependencies:
```bash
npm install
```

2. Set up your `.env` file at the root of the project with the following variables:
```env
DATABASE_URL="postgresql://postgres:admin@localhost:5432/waterlevel?schema=public"
NEXTAUTH_SECRET="supersecret123456"
NEXTAUTH_URL="http://localhost:3000"
```

3. Sync Prisma with the database and apply the schema:
```bash
npx prisma db push
```
*(If you run into locking errors on Windows, run `npx prisma generate` first or close the dev server during push).*

4. Seed the database with the initial admin account and dummy waterlevel data:
```bash
node seed.js
```

5. Run the development server:
```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) with your browser to see the result. You can log in using the default admin credentials generated by the seed script. You will be redirected to the login page if not authenticated.
